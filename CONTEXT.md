# Luoome 领域语言

## 核心概念

### 股票分组（StockGroup）

回答“看哪些股票”。分组维护成员集合，可来自手工列表、账户持仓、战法筛选或 LLM
筛选。动态来源通过成员快照与盘中盯盘解耦，因此同一个分组可以被多个场景复用。

### 盯盘方案（WatchPlan）

回答“这些股票发生什么时提醒”。一个方案引用一个股票分组，并配置一组触发规则、通知冷却
和启停状态。代码与持久化层暂时沿用 `StockPool` 名称以保持兼容；面向用户的 Web 与文档使用
“盯盘方案”。

Web 中不设独立“盯盘”信息孤岛：方案在其所属分组详情中配置；跨分组的运行心跳、手动试跑
和最近触发放在仪表盘。

公式分组使用的战法是成员生产条件，方案中的战法规则是盘中触发条件。两者职责不同，
允许引用不同的战法。

### 盯盘触发（WatchTrigger）

某只分组成员命中方案规则后产生的可审计事实，包含方向、理由、证据、行情快照和通知状态。

## 关键约束

- 停用的分组不再向盯盘方案提供成员，即使数据库里仍有旧快照。
- 动态分组刷新失败或空结果时保留旧快照，并通过 `stale` 状态提示用户。
- 盯盘方案不拥有分组成员；删除被方案引用的分组会被拒绝。
- 建议与真实交易严格分离，盯盘触发不会自动下单。
